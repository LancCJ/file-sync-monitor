# FileSyncMonitor 功能清单与细节文档

> 版本：v1.0  
> 适用平台：macOS 14+  
> 核心用途：监控指定目录的文件改动，记录所有变更，并提供未同步提醒、报告导出及同步状态手动重置功能。

---

## 一、核心功能总览

| 功能模块           | 简要说明                                                               |
| ------------------ | ---------------------------------------------------------------------- |
| 目录监控           | 支持监控一个或多个用户选择的目录（递归），实时捕获文件/文件夹的变动。   |
| 事件记录           | 将每次改动（创建、修改、删除、重命名）持久化存储，保留时间戳与路径信息。 |
| 未同步提醒         | 通过菜单栏图标、应用内列表、系统通知三种方式提醒用户哪些改动尚未同步。   |
| 状态重置           | 用户可以随时将任意改动标记为“已同步”或“未同步”，满足手动确认的需求。   |
| 报告导出           | 按日/周/月/年维度导出改动清单（CSV / JSON），便于审计或备份记录。       |
| 历史查看与筛选     | 在应用主窗口中浏览所有改动记录，支持按时间、路径、类型、同步状态筛选。   |

## 二、详细功能与细节说明

### 2.1 目录监控

| 细节项               | 说明                                                                                     |
| -------------------- | ---------------------------------------------------------------------------------------- |
| **监控目标**         | 任意用户指定的文件夹（不支持单文件监控，但文件夹内文件均会被递归监控）。                 |
| **监控能力**         | 实时捕获 `创建`、`修改`、`删除`、`重命名` 四种操作。                                     |
| **技术实现**         | macOS FSEvents API，低资源占用，支持历史事件去抖合并（例如同一文件 2 秒内多次修改只记一次）。 |
| **多目录支持**       | 允许同时添加多个目录，每个目录的改动独立记录，在列表中可区分来源。                       |
| **权限处理**         | 使用 `NSOpenPanel` 引导用户选择目录，并通过 Security-Scoped Bookmarks 持久化权限，应用重启后无需重新授权。 |
| **忽略规则（可选）** | 用户可配置忽略特定文件扩展名（如 `.tmp`、`.log`）或文件夹（如 `node_modules`），减少不必要的事件记录。 |

### 2.2 事件记录与数据模型

| 细节项               | 说明                                                                                     |
| -------------------- | ---------------------------------------------------------------------------------------- |
| **存储方式**         | CoreData + SQLite，本地数据库文件位于 `~/Library/Application Support/FileSyncMonitor/`。 |
| **记录字段**         | - 事件唯一 ID（UUID）<br>- 文件/文件夹绝对路径<br>- 原路径（仅重命名事件）<br>- 改动类型（created/modified/deleted/renamed）<br>- 发生时间（精确到秒）<br>- 同步状态（Bool，默认为 false）<br>- 是否已发送过通知（避免重复提醒） |
| **数据保留策略**     | 默认保留所有事件；可设置自动删除超过 N 天（例如 365 天）的**已同步**事件，未同步事件永不删除。 |
| **并发处理**         | 事件写入使用独立后台上下文，不阻塞 UI 和文件监控回调。                                   |

### 2.3 未同步提醒机制

| 提醒方式           | 细节说明                                                                                 |
| ------------------ | ---------------------------------------------------------------------------------------- |
| **菜单栏图标**     | 应用常驻菜单栏，图标上显示红色数字角标（当前未同步事件总数）。点击图标弹出下拉面板，显示最近 5 条未同步改动，并附“打开主窗口”和“全部标记同步”快捷操作。 |
| **主窗口内提醒**   | 侧边栏“待同步”条目显示未同步数量；主区域默认按时间倒序列出所有未同步事件，支持批量操作。 |
| **系统通知**       | 新事件产生时，若应用处于后台且该事件尚未触发过通知，则发送 macOS 本地通知（用户可全局关闭此功能）。通知内容包含文件路径和改动类型。 |
| **重复提醒策略**   | 未同步事件不会重复发送系统通知；但菜单栏角标会持续显示，直到用户手动标记同步或标记忽略。 |

### 2.4 状态重置（手动同步管理）

| 操作             | 说明                                                                                                   |
| ---------------- | ------------------------------------------------------------------------------------------------------ |
| **标记为已同步** | 在事件列表中，选中任意未同步事件，点击“✅ 标记已同步”后，该事件的 `isSynced` 变为 `true`，从未同步列表中消失，菜单栏角标数字减 1。 |
| **标记为未同步** | 选中任意已同步事件，点击“🔄 标记未同步”后，该事件重新进入未同步列表，菜单栏角标数字相应增加。           |
| **批量操作**     | 支持多选（Command + 点击 / Shift 范围选择），可批量标记同步或未同步。                                   |
| **使用场景**     | - 用户实际已同步云端，忘记点击 → 标记已同步消除提醒<br>- 用户误标记已同步 → 重新设为未同步继续提醒<br>- 用户想保留某个旧改动的提醒 → 重置为未同步 |
| **注意**         | 该操作**不会恢复文件内容**，仅改变提醒状态。                                                           |

### 2.5 报告导出功能

| 细节项             | 说明                                                                                         |
| ------------------ | -------------------------------------------------------------------------------------------- |
| **导出维度**       | 每日、每周、每月、每年。用户选择维度后，再指定具体日期（例如“2026年5月”代表该月的所有事件）。 |
| **时间区间计算**   | - 每日：指定日期的 00:00:00 至 23:59:59<br>- 每周：周一至周日（可根据系统区域调整）<br>- 每月：1 日至月末最后一天<br>- 每年：1 月 1 日至 12 月 31 日 |
| **导出内容**       | 包含字段：事件 ID、时间戳、文件路径、改动类型、同步状态（true/false）。                      |
| **支持格式**       | CSV（逗号分隔，UTF‑8 编码）、JSON（数组格式，每个事件为一个对象）。                          |
| **导出操作流程**   | 1. 在“报告中心”选择维度和日期<br>2. 点击“预览”可查看统计（如改动总数、未同步数量）<br>3. 点击“导出”后弹出保存面板，用户选择保存位置和格式。 |
| **文件命名示例**   | `FileSync_2026-05-15.csv` 或 `FileSync_2026_May.json`                                       |

### 2.6 主窗口 UI 与交互

| UI 区域         | 细节说明                                                                                     |
| --------------- | -------------------------------------------------------------------------------------------- |
| **侧边栏**      | - 所有改动：显示全部事件<br>- 待同步：仅显示 `isSynced == false` 的事件，带红色角标数字<br>- 报告中心：快捷跳转到导出页面<br>- 设置：监控目录管理、忽略规则、通知开关等 |
| **事件列表**    | 表格形式，列包含：时间、文件路径、改动类型、同步状态（图标显示）。支持点击列头排序。         |
| **筛选栏**      | 顶部提供过滤条件：<br>- 改动类型下拉（全部/创建/修改/删除/重命名）<br>- 同步状态（全部/已同步/未同步）<br>- 搜索框（按路径关键词模糊匹配） |
| **批量操作栏**  | 底部工具栏：<br>- “✅ 标记已同步”<br>- “🔄 标记未同步”<br>- “🗑️ 删除选中的事件记录”（谨慎使用） |
| **导出按钮**    | 任何视图均可在右上角找到“导出当前列表”按钮，快速导出当前筛选结果（不按时间维度，即所有可见记录）。 |

### 2.7 菜单栏下拉面板细节

- 图标状态：  
  - 未同步数 = 0 → 灰色勾图标，无数字  
  - 未同步数 > 0 → 红色圆白字数字  
- 下拉面板宽度 320pt，最大高度 400pt。  
- 内容：  
  - 标题行：“待同步 (X 个)” + “全部标记同步” 按钮  
  - 分割线  
  - 最多 5 条最新未同步记录（每条显示：图标表示类型 + 文件名 + 相对路径截断）  
  - 点击某条记录可直接将其标记为已同步（无需打开主窗口）  
  - 底部 “打开 FileSyncMonitor…” 按钮  

### 2.8 设置页面功能

| 设置项                 | 细节说明                                                                 |
| ---------------------- | ------------------------------------------------------------------------ |
| **监控目录管理**       | 列表显示当前监控的所有目录，支持添加（打开选择面板）和移除。              |
| **忽略规则**           | - 按扩展名忽略：例如 `.DS_Store`, `.tmp`<br>- 按文件夹名忽略：例如 `Cache`, `Trash`<br>- 正则表达式高级忽略（可选） |
| **通知设置**           | 开关：“新的改动时发送系统通知”                                            |
| **数据保留**           | 自动删除超过 X 天的已同步事件（默认 365 天，可改为 30/90/180/从不）       |
| **导出默认格式**       | 设置导出的默认格式（CSV 或 JSON）                                          |
| **重置所有状态**       | 将所有事件的 `isSynced` 重置为 `false`（谨慎操作，需要二次确认）          |

## 三、技术规格与限制

| 项目               | 规格                                                                 |
| ------------------ | -------------------------------------------------------------------- |
| 最低系统版本       | macOS 14.0 (Sonoma) 或更高                                           |
| 架构支持           | Apple Silicon (arm64) 和 Intel (x86_64)                              |
| 内存占用（典型）   | 约 50 MB (空闲) ~ 120 MB (大量事件加载)                              |
| 数据库存储估算     | 每个事件约 300 字节，10 万个事件约 30 MB                             |
| 最大监控文件数     | 无硬性限制，受系统 FSEvents 回调频率影响，建议单目录下不超过 200 万个文件 |
| 支持路径长度       | 最大 1024 字符（macOS PATH_MAX）                                     |
| 并发事件处理能力   | 每秒最多处理约 5000 次事件变化（实际取决于磁盘 IO）                   |

## 四、用户操作示例场景

### 场景 1：日常使用流程
1. 启动 FileSyncMonitor，添加监控目录 `/Users/me/Documents`。  
2. 编辑 `proposal.docx` 文件，应用立刻记录一条“修改”事件，菜单栏角标变为 **1**。  
3. 用户完成云端同步操作后，打开主窗口 → “待同步”列表，勾选该事件，点击“标记已同步”，角标消失。  

### 场景 2：周末集中同步
1. 一周内修改了 20 个文件，菜单栏角标显示 **20**。  
2. 周末时用户打开“待同步”列表，确认所有文件都已手动上传至自己的云盘。  
3. 全选列表（Cmd+A），点击“标记已同步”，全部清除。  

### 场景 3：导出月度报告
1. 打开“报告中心”，选择维度“每月”，选择“2026年5月”。  
2. 点击“预览”看到 5 月共有 45 次改动，其中 10 次未同步。  
3. 点击“导出”，选择 CSV 格式，保存到桌面，用于团队审计。  

### 场景 4：纠正误标记
1. 用户不小心把一个本应提醒的文件标记为“已同步”。  
2. 切换到“所有改动”视图，筛选“已同步”状态，找到该事件。  
3. 选中它，点击“标记未同步”，该事件重新出现在待同步列表中，角标数字恢复。

## 五、已知限制与未来规划

### 当前版本限制
- 不支持监控外部磁盘（U 盘、网络驱动器）的断开重连场景（断开期间的事件会丢失）。  
- 不支持文件内容的差异比对，仅记录“修改”事件。  
- 无法监控移动硬盘的弹出后重新插入时的历史变化。  
- 重命名事件记录中，如果重命名后路径过长可能导致 `oldFilePath` 截断（概率极低）。  

### 未来版本可能增加的功能
- 云存储原生集成（Google Drive, Dropbox）自动同步状态。  
- 文件快照功能：定期比对内容哈希，发现内容变化时记录。  
- 多设备同步监控状态（通过 iCloud）。  
- 支持导出 PDF 格式报告。  
- 快捷键支持（Cmd+M 标记同步等）。

## 六、附录：术语表

| 术语         | 解释                                                                 |
| ------------ | -------------------------------------------------------------------- |
| 未同步事件   | `isSynced == false` 的改动记录，提醒用户尚未确认同步。               |
| 已同步事件   | `isSynced == true` 的记录，表示用户已人工确认“该改动已同步到其他存储”。 |
| 状态重置     | 指将事件的 `isSynced` 从 true 改为 false 或相反，不涉及文件数据恢复。 |
| 安全作用域书签 | macOS 沙盒机制下持久化文件访问权限的方法。                           |
